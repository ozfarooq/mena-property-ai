Your First Working AI Agent
Claude API · Node.js · Tool use · Agentic loops
BUILD + SHIP
What you build this week
Project 1
Dubai Property Lead Qualifier Agent
A Node.js script that takes a raw property inquiry (text), uses Claude API with tool use to extract intent, budget, location preference, and buyer type, then outputs a structured JSON lead record. No UI needed — this is the core intelligence you'll wire into Make.com next week. This is a real, deployable thing a Dubai brokerage would pay for.
Stack
Claude API Tool Use / Function Calling Node.js @anthropic-ai/sdk dotenv
Build steps
01
Set up project + install SDK
npm init -y && npm install @anthropic-ai/sdk dotenv. Create a .env file with your ANTHROPIC_API_KEY. Test with a simple "hello" message call to verify connection.
docs.anthropic.com/getting-started →
02
Read: Tool Use (Function Calling) docs — full
This is the single most important concept for agentic AI. Tool use lets Claude call functions you define — like querying a database, sending a message, or extracting structured data. Read every example. ~45 min.
docs.anthropic.com/tool-use →
03
Build the lead extraction tool
Define a tool called extract_lead with parameters: intent (buy/rent/invest), budget_aed (number), locations (array), buyer_type (individual/investor/company), urgency (immediate/3months/exploring). Feed it sample WhatsApp-style inquiries in Arabic and English. Verify the JSON output is consistent.
04
Add an agentic loop
Extend it: if the inquiry is too vague (budget missing, no location), Claude should call a request_clarification tool instead — returning a follow-up question. This is your first agentic decision loop: the model decides which tool to call based on input quality. Read the "Agentic AI" section in Anthropic docs.
docs.anthropic.com/agentic-ai →
05
Test with 10 real-world inquiry samples
Write 10 sample inquiries yourself — mix of English and Arabic-style phrasing, varying levels of detail, different intents. Run all 10 through your agent. Note where it fails. Fix the system prompt. This is prompt engineering in practice, not theory.
The one concept to internalize this week: An "agentic loop" is when the LLM decides what tool to call next based on its own output. It's not a chatbot that answers questions — it's a decision-maker that takes actions. Your lead qualifier decides: "do I have enough data to extract, or do I ask a follow-up?" That decision loop is the foundation of every agent workflow you'll build.

Make.com + Claude — First Automation
Make.com · HTTP modules · Webhooks · CRM push
NO-CODE + AI
What you build this week
Project 2
End-to-End Lead Qualification Pipeline
Wire your Week 1 agent into Make.com. The flow: new form submission or email arrives → Make.com webhook triggers → HTTP module calls your Claude agent → structured lead JSON → pushed to Google Sheets (or Airtable) as a qualified lead record, with an automatic WhatsApp reply sent to the prospect. This is a complete, sellable automation.
Stack
Make.com HTTP Module (Claude API) Webhooks Google Sheets WhatsApp Business API JSON parsing
Build steps
01
Make.com crash course — official tutorials (3 hrs)
Watch Make.com's getting started series. Understand: scenarios, modules, webhooks, filters, data stores, error handling. Don't build yet — just watch and understand the visual programming model.
make.com/tutorials →
02
Build webhook → Claude API HTTP call
Create a Make.com webhook that accepts a POST body with an inquiry string. Use the HTTP module to call Anthropic's API directly (POST to https://api.anthropic.com/v1/messages with your auth header). Parse the tool_use response to extract the lead JSON. This is the core skill — calling any AI API from Make.com.
03
Push qualified lead to Google Sheets
Map the extracted JSON fields to a Google Sheet: timestamp, name (if in inquiry), intent, budget, location, urgency, raw_message. Add a "lead_score" column — high/medium/low based on budget threshold. This is the CRM layer.
04
Auto-reply via WhatsApp Business
If lead_score is high, trigger a personalized WhatsApp message back to the prospect using the WhatsApp Business Cloud API. The message is generated by Claude — not a template. Pass the lead details back to Claude and ask it to write a warm, professional first response in the prospect's language.
05
Record a 3-minute Loom demo of the full flow
Trigger the webhook, watch the lead appear in the sheet, show the WhatsApp reply. This Loom video is your first sales asset. Post it on LinkedIn this week with the caption: "I built this for Dubai real estate brokerages. Inquiry to qualified lead in under 8 seconds, 24/7."
Make.com is not a toy. It's how you deliver AI agent workflows to non-technical clients without writing a full backend. The HTTP module means you can call any API — Claude, OpenAI, WhatsApp, Supabase, any webhook — from a visual flow. Most of your UAE/Saudi clients will never see code. They'll see the Make.com scenario and the results it produces.

WhatsApp AI Agent — Full Conversation
n8n · Multi-turn memory · Conversation state · Arabic/English
FULL AGENT
What you build this week
Project 3
Conversational WhatsApp Property Agent
A full multi-turn WhatsApp agent that handles ongoing property conversations — not just single messages. Built in n8n. The agent remembers what was said earlier in the conversation, handles Arabic and English naturally, answers property FAQs, qualifies the lead over multiple messages, and escalates to a human broker when ready. This is Zara v2 — productized for any real estate client.
Stack
n8n WhatsApp Business Cloud API Claude API (messages array) Redis or n8n data store (memory) Webhook triggers Conditional routing
Build steps
01
Set up WhatsApp Business Cloud API
Create a Meta developer account, set up a test WhatsApp Business number, configure a webhook in n8n to receive incoming messages. Test by sending a message to your test number and watching it appear in n8n. This is the plumbing — get it working before building any logic.
developers.facebook.com →
02
Build conversation memory with n8n data store
The key challenge in multi-turn agents: the LLM has no memory between API calls. You need to store and retrieve the conversation history. Use n8n's built-in data store (or a simple JSON file keyed by WhatsApp number) to persist the messages array. On each new message: load history → append new user message → call Claude → append response → save back.
03
Write the system prompt for the property agent
This is your Week 1 prompt engineering payoff. Write a production system prompt that: defines the agent persona, sets response language rules (detect Arabic → respond Arabic), defines qualification goals, specifies escalation triggers (when to say "let me connect you with an agent"), and sets tone. Use XML tags for structure.
04
Add human escalation routing
When the lead is qualified (or asks for a human), the agent should: send a summary message to the prospect ("Our agent X will contact you within 2 hours"), post a notification to a broker's WhatsApp or Telegram with the full lead summary. This is the handoff pattern every real estate client needs.
05
Test with 5 full conversations (different personas)
Play 5 different prospect personas: an overseas investor, a tenant looking to rent, a vague inquiry, a very specific buyer, someone writing in Arabic. Put the agent under pressure. Fix conversation breaks. Document the failure patterns — you'll need this for client demos.
Why this week matters for income: This is the most in-demand AI product in the UAE real estate market right now. Every brokerage with 5+ agents and active WhatsApp inquiries is a potential client. The conversation memory pattern you learn here applies to every other vertical: healthcare appointment booking, restaurant reservations, customer support. One pattern, unlimited clients.

Multi-Agent Workflow
Orchestrator + specialist agents · n8n subflows · Agent-to-agent handoff
ADVANCED
What you build this week
Project 4
Property Intelligence Orchestrator
A multi-agent system where one orchestrator agent coordinates three specialist agents: (1) a Lead Qualifier that extracts intent from inquiries, (2) a Market Analyst that pulls comparable properties and yields from DXBinteract data, (3) a Report Writer that generates a personalized investor brief in English or Arabic. The orchestrator decides which agents to call and in what order. This is the "agentic AI" pattern the market is paying for.
Stack
Claude API (orchestrator) n8n subflows (specialist agents) Tool use (agent routing) DXBinteract or mock data API PDF or HTML report generation Make.com (delivery)
Build steps
01
Read: Anthropic multi-agent patterns docs
Understand orchestrator-worker architecture. The orchestrator is a Claude instance with tools that call other agents. It doesn't do the work — it decides what work needs doing and delegates. This is exactly how large enterprise automation systems work.
docs.anthropic.com/agentic-ai →
02
Build the 3 specialist agents as n8n subflows
Each subflow accepts a JSON input, does one specific job, and returns a JSON output. Keep them dumb and focused — the orchestrator is what makes them smart. Lead Qualifier: you built this in Week 1. Market Analyst: call a mock data endpoint (or DXBinteract if you get API access) and summarize comparable listings. Report Writer: given lead + market data, write a 300-word investor brief.
03
Build the orchestrator with tool use
Create a Claude orchestrator that has 3 tools defined: call_lead_qualifier, call_market_analyst, call_report_writer. Each tool triggers the corresponding n8n subflow via HTTP. Write a system prompt that tells the orchestrator: given a raw inquiry, decide which agents to call in what order to produce a complete investor brief.
04
Generate a PDF report and email it
Take the final report text, generate a basic HTML → PDF using a tool like Puppeteer (Node.js) or Make.com's HTML-to-PDF module. Email it to the prospect automatically via Gmail module in Make.com. Subject line generated by Claude based on the inquiry content.
Why multi-agent matters for your positioning: Single-prompt automations are commoditized. Multi-agent workflows that coordinate specialized models to produce complex, multi-step outputs are where consulting rates live — $3K–8K per deployment, not $500. Understanding the orchestrator-worker pattern puts you in a different conversation with clients than someone who builds simple chatbots.

What you build this week
Project 5
Knowledge-Grounded Property Agent
Upgrade your Week 3 WhatsApp agent with a RAG layer. Feed it a knowledge base: RERA regulations, community guides for Dubai Marina / JVC / Downtown, developer payment plan FAQs, DIFC visa rules. The agent now answers complex questions accurately from your knowledge base — not from LLM hallucination. You already built the RAG pipeline (your dev.to article). Stack
Supabase pgvector (you already know this) Voyage-3 embeddings Claude API (generation) n8n (orchestration) Markdown knowledge base files
Build steps
01
Build the knowledge base (the real work)
Write or collect 15–20 markdown documents covering: RERA rental index rules, Ejari process, common community profiles (JVC, JBR, Downtown, Business Bay), off-plan buying process, Golden Visa via property, DLD transfer process, service charge ranges by community. This content is your product's defensible edge — not the code.
02
Ingest into Supabase — reuse your RAG pipeline
You already wrote the ingestion script in your dev.to article. Run it on your knowledge base files. Verify embeddings are stored. Test similarity search with 5 property-related queries. Aim for similarity scores above 0.78 on relevant chunks.
03
Add RAG retrieval step to n8n agent flow
Before calling Claude with a user question, add an n8n step: embed the question → run similarity search → retrieve top 3 chunks → inject into system prompt as context. The agent now answers from your knowledge, not from training data. Test with RERA-specific questions that would stump a general LLM.
04
Add a "knowledge gap" detection tool
If similarity scores are all below 0.65, the agent doesn't have relevant knowledge. Instead of hallucinating, it should say: "I don't have specific information on that — let me connect you with an agent who can help." Add this check as a conditional in n8n. Never let your agent make things up.
